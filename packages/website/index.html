Features

One job, done exceptionally well

Mergenote does not try to be a platform. It turns merged pull requests into release notes you would be happy to publish.

Grouping that gets it right

Mergenote reads PR titles, labels, and conventional commit prefixes, then sorts everything into Added, Fixed, Changed, and Internal. No manual triage on release day.

  • Reads labels and conventional commits
  • Filters noise like merge and bot commits
  • Works by tag range or date range

Quickstart

Connected in under a minute

Add one block to your MCP config, restart your client, and ask for your first changelog.

1

Add Mergenote to your client

Drop the block into mcp.json for Antigravity, Claude Code, Cursor, or any MCP client.

2

Add your GitHub token

A read scoped token lets Mergenote list merged pull requests for your repo.

3

Ask for a changelog

Say generate release notes since v2.3.0 and review the result inline.

mcp.json
{
  "mcpServers": {
    "mergenote": {
      "command": "npx",
      "args": ["-y", "mergenote"],
      "env": {
        "GITHUB_TOKEN": "ghp_xxx",
        "MERGENOTE_LICENSE_KEY": "mn_live_xxx"
      }
    }
  }
}

FAQ

Questions, answered

What is an MCP server?

The Model Context Protocol is an open standard that lets AI clients call external tools. Mergenote runs as an MCP server, so any compatible client like Antigravity, Claude Code, or Cursor can use it.

Which clients does Mergenote work with?

Any client that speaks MCP. That includes Antigravity, Claude Code, Cursor, and others. You add one block to the client config and restart.

Do you store my code?

No. Mergenote reads pull request metadata with your own GitHub token and can generate summaries on your client model. Your source code is never stored on our servers.

How does billing work?

Checkout runs through PayPal. After payment you get a license key that unlocks Pro features in any client. Cancel anytime and you keep access until the period ends.

Can I customize the format?

Yes. Use the Keep a Changelog preset or define your own sections, headings, and tone. Pro plans save a template per repository so every release stays consistent.
